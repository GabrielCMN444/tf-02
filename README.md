TF002 – Arquitetura em Camadas
Sobre o trabalho

Este repositório contém a atividade TF002 – Desenhando a “Planta” de um Sistema, realizada para a disciplina de Implementação de Servidor e Nuvem (Cloud).

O objetivo da atividade é entender como funciona a arquitetura em camadas (3-Tier) e como as responsabilidades de um sistema são separadas entre:

Camada de Apresentação

Camada de Negócio

Camada de Dados

Sistema analisado

Para explicar as camadas, foi utilizado como exemplo um aplicativo de mensagens semelhante ao WhatsApp.

A partir desse exemplo, foram identificados:

elementos da interface que o usuário vê

regras de funcionamento do sistema

informações que precisam ser armazenadas

Estrutura do trabalho

O trabalho foi dividido em quatro partes:

Parte 1 – Mapa das Camadas
Identificação dos elementos de apresentação, regras de negócio e dados.

Parte 2 – Análise de impacto
Explicação do que acontece quando alguma parte do sistema muda ou deixa de funcionar.

Parte 3 – Identificação de erros
Análise de situações em que as camadas estão misturando responsabilidades.

Parte 4 – Analogia
Explicação do conceito utilizando um exemplo do mundo real.

Objetivo

O objetivo principal é compreender como a separação em camadas ajuda a tornar sistemas mais organizados, seguros e fáceis de manter.
